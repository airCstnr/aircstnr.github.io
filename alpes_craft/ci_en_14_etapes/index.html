L'intégration continue en quatorze étapes

Alpes Craft 2023

Speaker: Thierry de Pauw

Introduction

L'intégration continue est un ensemble de pratiques qui permettent de livrer rapidement un produit logiciel de qualité.

Nous allons présenter ici 14 pratiques essentielles à l'intégration continue.

I. Conventions d'équipe

  1. Tout placer sous gestionnaire de version
  2. Accepter en tant qu'équipe de ne jamais casser le Build
  3. Ne pas pousser sur un Build cassé
  4. Revert quand le Build est cassé

II. Code

  1. Faire tous les changements en petits incréments
  2. Commiter fréquemment
  3. Commiter seulement quand les tests passent
  4. Découpler le code
  5. Utiliser le pattern Expand-Contract
  6. Cacher les fonctionnalités qui ne sont pas terminées

III. Construction

  1. Automatiser le Build
  2. Lancer le Build localement
  3. Avoir un grand nombre de tests automatisés de haute qualité
  4. Avoir un Build rapide

Conclusion

Seulement 2 outils sont nécessaires à toutes les pratiques de l'intégration continue :

  • un gestionnaire de versions
  • un script de construction qui prend moins de 10 minutes
